Creating installers for Shiny apps on Windows

2022/02/19

I had the goal of packaging a Shiny app such that I could simply hand an installation file to a person, they install the file on their system and they can use the Shiny app. Most common way of sharing Shiny apps is uploading the app online. But if the app requires raw data, there are good reasons why people should not upload their raw data.

The setup I lay out is for Windows, but it should be possible to do all this for Mac. The main difference is you need R-portable for Mac. And if you search the internet enough, you will find an R-portable for Mac.

R-Portable

First thing is we need a portable version of R bundled with the installer that the app can rely on. That way, we’re unconcerned about the user having (the required version of) R on their machine. Alternatively, we could have a setup where as part of the installation, we install R on the user’s system, but that could mess up their machine’s R setup.

R-portable is available here: https://portableapps.com/node/32898

Workflow: Basic vs. Electron

There are two basic workflows for creating an installer. We can take the bare-bones options, or use Electron. Bare-bones option is simple enough and has less hassle so I will cover it here. Electron is a framework for building cross-platform apps. I was able to get Electron to work, but I find it unnecessary. I will not cover it here, but if you find yourself wanting to use Electron and are struggling, you can always contact me.
